# YOLLR Documentation Package (ZIP-READY)

This directory contains the complete foundational documentation needed to plan, design, and build the full Yollr platform — including the AI builder prompt, PRD, system architecture, database specification, UI/UX specifications, and all flows.

## Folder Structure

```
/prompt
   yollr-ai-builder-prompt.md

/product
   yollr-product-requirements.md

/technical
   yollr-technical-specification.md
   yollr-ui-ux-specification.md
```

## Purpose of Each File

### 1. yollr-ai-builder-prompt.md
A master prompt built for Claude Code, GPT, Cursor, or any modern AI code generator.  
Use this to bootstrap the entire app in automated or semi-automated code-generation tools.

### 2. yollr-product-requirements.md
A complete product requirements document describing the “why” and “what” of the Yollr system — including user needs, audience analysis, and system-wide behavior.

### 3. yollr-technical-specification.md
Full technical blueprint including:
- Architecture
- Database schema
- RLS rules
- API design
- Feature logic
- Weekly state machine
- PWA spec
- Performance models
- Security architecture
- YoBeast template engine

### 4. yollr-ui-ux-specification.md
Complete UX/UI specification including:
- Figma file structure
- All 16 screens
- All flows
- Components library
- Color + typography systems
- Motion design
- Accessibility
- Visual standards

## Usage
Developers → set up repo, build feature modules  
Designers → build Figma from UI/UX spec  
Architects → review system foundation  
Founders → share with investors, pitch partners, schools
